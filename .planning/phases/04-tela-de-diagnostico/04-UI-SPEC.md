| Token | Value | Usage |

- `.data-row` existing class — `py-2` (8px) minimum touch row height. Phase 4 does not modify this class.

Source: Diagnosis.jsx existing layout (`space-y-4`, `p-5`, `p-6`), index.css `.data-row`

## Typography

Exactly 4 sizes, 2 weights:

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Label | 14px | `text-sm` | 700 (bold) | 1.4 | Section micro-labels, badge text, data-row labels. Micro context uses `uppercase tracking-wider` modifier — same 14px base, no separate size. |
| Body | 15px | `text-[15px]` | 400 (regular) | 1.55 | Global body, diagnosis markdown paragraphs |
| Heading | 20px | `text-xl` | 700 (bold) | 1.3 | Card headings, projection values, section subheads |
| Display | 36px | `text-4xl` | 700 (bold) | 1.1 | Lucro Líquido primary value AND business name H1. Covers both primary value and heading-hero roles. |

Additional rules:
- All monetary values: `font-mono` (JetBrains Mono) + `tracking-tighter` + `tabular-nums`. Never Inter for numbers.
- Section micro-labels: `text-sm font-bold uppercase tracking-wider` — color is `text-ink-400` (light surface cards) or `text-ink-300` (dark `.card-dark`). No `text-xs` — use `text-sm` (14px) for all micro-labels.
- Diagnosis markdown h2: `text-[15px] font-bold text-ink-800` (existing `.diagnosis-content h2` rule — keep unchanged).
- Number count-up animation: animated value uses same `font-mono text-4xl font-bold` as static value. Animation is purely numeric interpolation — no font change.
- New animated number role (DG-01): counters on entry use `useEffect` + `requestAnimationFrame` or a Framer Motion `useMotionValue` interpolated from 0 to target over 1200ms with `easeOut` easing.
- ProjectionCard projected value: `text-xl font-bold tracking-tighter font-mono` (20px Heading) — do NOT use `text-2xl` (24px).
- DiagnosisHeader business name: `text-4xl font-bold text-ink-900 tracking-tighter` (36px Display) — do NOT use `text-3xl` (30px).

Source: index.css `body { font-size: 15px; line-height: 1.55 }`, `.diagnosis-content`, tailwind.config.js `fontFamily.mono`, Diagnosis.jsx `text-4xl font-bold font-mono`

## Animation Contract

All animations use Framer Motion. Variants declared at module scope outside components.

```js
// Module-scope variants — declare once, reuse across all sections
const fadeUp = {
  hidden:  { opacity: 0, y: 16 },
  visible: { opacity: 1, y: 0, transition: { duration: 0.5, ease: [0.16, 1, 0.3, 1] } },
};

const shakeIn = {
  hidden:  { opacity: 0, x: -8 },
  visible: { opacity: 1, x: 0, transition: { type: 'spring', stiffness: 300, damping: 20 } },
};

const staggerContainer = {
  hidden:  {},
  visible: { transition: { staggerChildren: 0.1, delayChildren: 0 } },
};
```

### DG-02: Stagger Reveal

Wrap all Diagnosis sections in a `motion.div` with `variants={staggerContainer}` + `initial="hidden" animate="visible"`.
Each card/section is a `motion.div` with `variants={fadeUp}` (or `shakeIn` for alert cards).

| Section | Variant | Stagger delay |
|---------|---------|---------------|
| DiagnosisHeader | fadeUp | 0s (delayChildren base) |
| HealthBadge | fadeUp | +0.1s |
| AlertCard (mixed accounts) | shakeIn | +0.15s |
| HeroCard (Lucro Líquido) | fadeUp | +0.2s |
| DiagnosisMarkdownCard | fadeUp | +0.3s |
| BenchmarkChart | fadeUp | +0.4s |
| BenchmarkPremium | fadeUp | +0.5s |
| BreakEvenCard | fadeUp | +0.55s |
| ProjectionCard | fadeUp | +0.6s |
| ExportCard | fadeUp | +0.7s |
| ActionButtons group | fadeUp | +0.8s |

### DG-01: Number Count-Up

Applies to: Lucro Líquido, Lucro Bruto, EBITDA (HeroCard), Ponto de Equilíbrio (BreakEvenCard), Projeção 30 dias (ProjectionCard).

Implementation pattern (single hook, reuse across all):
```js
function useCountUp(target, duration = 1200) {
  const [value, setValue] = useState(0);
  useEffect(() => {
    let start = null;
    const step = (timestamp) => {
      if (!start) start = timestamp;
      const progress = Math.min((timestamp - start) / duration, 1);
      const eased = 1 - Math.pow(1 - progress, 3); // easeOutCubic
      setValue(target * eased);
      if (progress < 1) requestAnimationFrame(step);
    };
    requestAnimationFrame(step);
  }, [target, duration]);
  return value;
}
```

- Apply `useCountUp` to each numeric field value.
- Format with `formatBRL(value)` (existing util — already handles negative).
- Duration: primary value 1200ms, secondary values 1000ms, projection 900ms.
- Do NOT animate percentage values with count-up (netMargin, grossMargin) — use static display. Percentages are secondary context, not headline metrics.
- Trigger: count-up starts on component mount (when Diagnosis screen becomes visible). No scroll trigger needed — single-screen component.

### DG-03: Alert Card Differentiation

Alert cards (HealthBadge with `warn`/`loss` tone, AlertCard for mixed accounts):
- Use `shakeIn` variant (x-axis spring entrance) instead of `fadeUp`
- Add pulsing status dot: `animate-pulse` on the `w-2.5 h-2.5 rounded-full {tone.dot}` element
- Add left border accent: `border-l-4` with tone-appropriate color
- Add `shadow-md` (elevated shadow) vs normal cards which use `shadow-card`

Normal data cards (HeroCard, BenchmarkChart, etc.): use `fadeUp` variant, `shadow-card`.

### Screen Entry Transition

When App.jsx transitions from `LOADING` → `DIAGNOSIS` step, the entire Diagnosis screen enters via:
- `motion.div` wrapping `<Diagnosis>` in App.jsx render: `initial={{ opacity: 0 }} animate={{ opacity: 1 }} transition={{ duration: 0.4, ease: [0.16, 1, 0.3, 1] }}`
- This outer fade is additive to the staggerContainer inside — outer fade takes 0.4s, inner stagger starts immediately, creating a layered reveal.

`prefers-reduced-motion`: existing `@media (prefers-reduced-motion: reduce)` in index.css collapses all animations. Framer Motion respects this via `useReducedMotion()` — add check: if `shouldReduceMotion`, skip stagger delays and count-up animations.
