## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (gap-1, gap-1.5) |
| sm | 8px | Compact element spacing (gap-2, p-2) |
| md | 16px | Default element spacing (px-4, py-4, gap-4) |
| lg | 24px | Section internal padding (px-6, py-6, gap-6) |
| xl | 32px | Layout gaps between groups (gap-8, mt-8) |
| 2xl | 48px | Major section breaks (py-12, mt-12) |
| 3xl | 64px | Page-level section spacing (py-16, py-20) |

Exceptions:
- Touch targets (CTA buttons): minimum 56px height (`py-3.5 px-7` = 56px tall)
- Navbar height: 64px fixed (`h-16`)
- Hero section: `min-h-screen` (100dvh)
- ReportCard floating badge: uses float animation with `translateY(0 ‚Üí -8px)` ‚Äî 8px oscillation from tailwind `float` keyframe

Source: existing index.css `.btn-gold` (py-3.5 px-7), `.landing-container` (px-6), CONTEXT.md D-01 through D-12

## Component Inventory

All components below are built from existing project primitives. No new installs.

### Navbar
- Position: `fixed top-0 left-0 right-0 z-50`
- Background: `backdrop-blur-md bg-navy-950/80 border-b border-white/10`
- Height: 64px (`h-16`)
- Contents: Logo (left) | Nav links (center, hidden mobile) | CTA button (right, `.btn-gold` compact)
- Scroll behavior: always visible ‚Äî stays glass on scroll (no hide/show)
- Logo update: `money-500` icon square ‚Üí `gold-500` (`bg-gold-500`)

### Hero Section
- Full viewport: `min-h-screen` with `position: relative overflow-hidden`
- Background: slate-950 (`#0F172A`) base via `tailwind.config.js` or navy-900 ‚Äî 2 absolute `div`s with `radial-gradient(gold-500/gold-400)` animated by `glow-pulse` keyframe
- Animation: `glow-pulse` keyframe in `tailwind.config.js > keyframes` ‚Äî `scale(1‚Üí1.15) + opacity(0.08‚Üí0.15)`, duration 8‚Äì12s, ease-in-out, alternate direction, 2 divs offset timing by 4s for asynchrony
- Entry sequence (Framer Motion staggerContainer, delayChildren 0.1):
  1. Badge "Diagn√≥stico gratuito" ‚Äî `fadeUp` 0ms
  2. Headline line 1 ‚Äî `fadeUp` stagger 0.15s
  3. Headline line 2 ‚Äî `fadeUp` stagger 0.15s
  4. Subtext ‚Äî `fadeUp` stagger 0.15s
  5. CTA buttons row ‚Äî `fadeUp` stagger 0.15s
- ReportCard: absolute right side (hidden mobile), scroll-driven reveal via `useScroll({ target: heroRef })` + `useTransform` ‚Äî each card section (header ‚Üí badge ‚Üí metrics ‚Üí recommendation) maps to scrollYProgress thresholds: 0‚Üí0.1, 0.1‚Üí0.2, 0.2‚Üí0.35, 0.35‚Üí0.5
- Floating badge "Voc√™ est√° acima da m√©dia": `animate-float` (existing keyframe, 6s ease-in-out)

### Pain Points Section
- Layout: 2-column grid (mobile: 1-col), 6 items from `PAIN_POINTS` array
- Card style: `.glass-card` ‚Äî subtle, empathetic tone
- Each item: icon (gold-400) + heading + 1-line description
- Scroll trigger: `InView` component with `margin: '-60px'`, staggerChildren 0.07s

### "Como Funciona" Section (3 Steps)
- Layout: 3-column horizontal (mobile: vertical stacked), numbered steps 01/02/03
- Step number: 40px, gold-400, font-bold, letter-spacing tightest
- Connector line between steps: visible desktop only, `border-t border-white/10`
- Entry: staggerChildren 0.12s via `InView`

### Features Section (6 Cards)
- Layout: 3-column grid (mobile: 1-col, tablet: 2-col)
- Card style: `.card-navy` with hover `border-gold-500/30` transition
- Icon: inline SVG, gold-400, 24px
- Scroll trigger: staggerChildren 0.07s

### FAQ Section
- Component: custom accordion ‚Äî each item is a `<details>`-like pattern using Framer Motion `AnimatePresence` + `motion.div` height animation
- State: single open at a time (controlled via `useState`)
- Open indicator: chevron rotates 180deg (`rotate-180`)
- Divider: `border-b border-white/10` between items
- Animation: `slideUp` variant on content reveal

### Footer
- Layout: 3 columns (logo+tagline | links | legal)
- Background: `navy-950` or `#030810` (darkest)
- Border top: `border-t border-white/10`
- Links: `text-white/50` ‚Üí `text-white/80` on hover

Source: CONTEXT.md D-09/D-10/D-11/D-12, REQUIREMENTS.md LP-01 through LP-09, index.css existing patterns

## Animation Contract

All animations use Framer Motion. Variants are declared outside components (performance pattern from CONTEXT.md D-07).

| Element | Variant | Trigger | Duration | Notes |
|---------|---------|---------|----------|-------|
| Hero badge/headline/CTA | `fadeUp` staggerContainer | mount | 0.5s per element, 0.15s stagger | Reuse existing `fadeUp` variant |
| Scroll sections | `fadeUp` via `InView` | `useInView margin -60px` | 0.5s | once: true |
| Feature cards | staggerChildren 0.07s | `InView` | 0.5s | |
| ReportCard sections | `useScroll + useTransform` | heroRef scroll | ‚Äî | scrollYProgress thresholds per D-11 |
| Floating badge | `animate-float` | mount (after hero entry) | 6s infinite | Existing tailwind keyframe |
| Hero background glows | CSS keyframe `glow-pulse` | CSS (no JS) | 8‚Äì12s infinite alternate | Two divs, 4s offset |
| FAQ accordion | `AnimatePresence` height | click | 0.3s ease | slideUp variant |
| CTA hover | `active:scale-[0.98]` | pointer | 150ms | Existing .btn-gold pattern |

`prefers-reduced-motion`: all animations collapse to instant (existing `@media` rule in index.css already handles this).
